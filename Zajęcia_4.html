Bazy danych NoSQL

Not Only SQL, jak również nierelacyjne bazy danych.

  • klucz - wartość (key-value)
  • dokumentów - przechowywane są całe dokumenty, np kod w JSON reprezentujący wpis na blogu
  • serii czasu (time series) - mogą to być również relacyjne bazy ze szczególnym traktowaniem kolumn daty/czasu
  • kolumnowe - bazy gdzie jednostką przechowywania są kolumny, a czasem również grupy kolumn.
    • Niektóre bazy relacyjne również oferują kolumnowe formaty przechowywania, np SQL Server Mariadb
    • bazy typu column store cechuje większa wydajność dla dużych zapytań analitycznych, typowych dla hurtowni.
  • bazy grafów - zamiast sztywnego schematu przechowywana jest informacja o węzłach grafu i ich właściwościach oraz o krawędziech grafu i ich właściwościach.
    • spowodowały powstanie nowych języków zapytań: np Cypher dla bazy Neo4j
